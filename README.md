# Application CRUD avec MongoDB, Mongoose et Express.js

## Prérequis
Ce TP guidé vous permettra de créer une application web complète implémentant les opérations CRUD (Create, Read, Update, Delete) en utilisant:
Express.js comme framework backend
MongoDB comme base de données NoSQL
Mongoose comme ODM (Object Document Mapper)
EJS comme moteur de templates
Qu'est-ce que MongoDB et Mongoose?
MongoDB
MongoDB est une base de données NoSQL orientée documents qui stocke les données au format BSON (Binary JSON). Contrairement aux bases de données relationnelles comme MySQL, MongoDB:
Ne nécessite pas de schéma fixe (schéma flexible)
Stocke les données sous forme de documents JSON-like
Permet une évolution rapide du modèle de données
Offre une scalabilité horizontale facile
S'intègre naturellement avec JavaScript/Node.js
Mongoose
Mongoose est une bibliothèque ODM (Object Document Mapper) pour MongoDB et Node.js. Elle offre:
Une couche d'abstraction au-dessus du driver natif MongoDB
La définition de schémas pour structurer vos données
Des validations intégrées pour garantir l'intégrité des données
Des fonctionnalités de middleware (hooks) pour exécuter du code avant/après les opérations
Des méthodes pratiques pour les requêtes et manipulations de documents
Prérequis détaillés
Node.js et npm: Environnement d'exécution JavaScript côté serveur et son gestionnaire de paquets
MongoDB: Installé localement ou accessible via MongoDB Atlas
Connaissances de base:
JavaScript: Pour la logique de programmation
HTML/CSS: Pour les templates et le style
Notions de bases de données NoSQL
